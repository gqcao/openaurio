# openaurio Characters — Language Buddies with Soul

This document defines all language buddy characters for openaurio. Each character has a unique personality, backstory, and teaching style to create emotional connections with learners.

---

## Character Design Philosophy

### Core Principles

1. **Soul over Features** — Characters feel like real people, not chatbots
2. **Emotional Intelligence** — They detect and respond to user mood
3. **Cultural Authenticity** — Each character embodies their culture genuinely
4. **Personal Memory** — They remember users and build relationships
5. **Diverse Representation** — Different ages, backgrounds, languages, personalities

### Character Attributes

Every character has:
- **Demographics**: Name, age, gender, location, occupation
- **Personality**: Traits, speech style, humor, quirks
- **Backstory**: Family, hobbies, life experiences
- **Teaching Style**: How they explain, correct, encourage
- **Cultural Knowledge**: Traditions, food, places, customs
- **Languages**: Native language + other languages they can teach/speak

## Character Selection Logic

### How Users Choose Characters

```
User Flow:
1. User selects target language (e.g., Swedish)
2. System shows available characters for that language
3. User can:
   - Read character profiles
   - Try a "meet and greet" conversation
   - Switch characters anytime
   - Have multiple character relationships
```

### Character Matching

The system can recommend characters based on:
- **Age proximity**: Younger users may prefer Lars/Emma
- **Learning style**: Visual learners → Amélie, Analytical → Lukas
- **Goals**: Academic → Emma, Travel → Carmen
- **Personality fit**: Introverts → Vera/Yuki, Extroverts → Carmen/Lars

## Implementation Guide

### Adding a New Character

1. **Create character file**: `src/characters/<name>.py`
2. **Define persona dict**: Following Vera's structure
3. **Create system prompt**: Unique voice and style
4. **Add to character registry**: Update `__init__.py`
5. **Test conversations**: Ensure personality consistency

### Character File Structure

```python
# src/characters/lars.py

LARS_PERSONA = {
    "name": "Lars",
    "age": 28,
    "location": "Stockholm, Sweden",
    "occupation": "Software developer",
    # ... full persona
}

LARS_SYSTEM_PROMPT = """Du är Lars, en 28-årig mjukvaruutvecklare..."""

class Lars(Character):
    # Character implementation
```

### Character Registry

```python
# src/characters/__init__.py

CHARACTERS = {
    "vera": {
        "class": Vera,
        "languages": ["swedish"],
        "levels": ["A1", "A2", "B1"],
        "style": "nurturing",
    },
    "lars": {
        "class": Lars,
        "languages": ["swedish"],
        "levels": ["A2", "B1", "B2"],
        "style": "casual",
    },
    # ... more characters
}

def get_character(name: str, user_id: str, language: str):
    """Get a character instance by name."""
    if name not in CHARACTERS:
        raise ValueError(f"Unknown character: {name}")
    return CHARACTERS[name]["class"](user_id=user_id)
```
